int WSAStartup(WORD wVersionRequested, LPWSADATA lpWSAData);

​ 头文件<WinSock2.h>

​ 使用socket api之前需要通过该函数指定Socket版本，版本信息由第一个参数wVersionRequested传入，主版本号放于低字节，副版本号放于高字节，第二个参数为操作系统返回的Socket的信息，包含版本和描述信息等。

​ 执行成功后返回0，完成应用程序与socket库的绑定，应用程序便可访问其他socket api。

​ WSADATA介绍(win32)：

int WSACleanup();

​ 头文件<WinSock2.h>

​ 解除应用程序与socket的绑定，释放socket库所占用的系统资源。

int WSAGetLastError ();

​ 头文件<WinSock2.h>

SOCKET socket(int af, int type, int protocol);

​ windows头文件：<WinSock2.h>

​ linux头文件：<sys/socket.h>

​ 该函数用来创建并监听Socket，失败返回INVALID_SOCKET

  • af:指定通讯协议族
    • AF_INET,PF_INET: IPv4地址协议
    • PF_INET6：IPv6地址协议
  • type: 指定通讯的类型
    • SOCK_STREAM: Tcp连接，字节流
    • SOCK_DGRAM：Udp连接，数据报
  • protocol: 一般设置为0

int closesocket(SOCKET s);

​ windows头文件：<WinSock2.h>

​ linux头文件：<sys/socket.h>

​ 关闭socke，执行成功返回0，否则返回SOCKET_ERROR

int bind(SOCKET s, const struct sockaddr FAR *name, int namelen);

​ windows头文件：<WinSock2.h>

​ linux头文件：<sys/socket.h> <netinet/in.h>

​ 绑定ip和端口，一般服务端使用，客户端不使用，第二个参数用来指定ip和端口号，第三个参数用来指定第二个参数的大小，最好传入参数原型的大小

​ 执行成功返回0，否则返回SOCKET_ERROR

sockaddr也可以用来绑定ip和端口，但是其将ip与端口信息混在了一起，所以最好使用sockaddr_in：

int listen(SOCKET s, int backlog);

​ windows头文件：<WinSock2.h>

​ linux头文件：<sys/socket.h>

​ 使socket处于监听状态，处于监听状态的套接字s将维护一个客户连接请求队列，最多可容纳backlog个连接请求。

​ 执行成功返回0，失败返回SOCKET_ERROR.

SOCKET accept(SOCKET s, struct sockaddr FAR *addr, int FAR *addrlen);

​ windows头文件：<WinSock2.h>

​ linux头文件：<sys/socket.h>

​ 从listen维护的请求队列中取出排在最前的请求，并创建连接通道，如果连接成功就，返回SOCKET对象保存对方的socket信息，失败则返回INVALID_SOCKET.

​ 第二个参数为对方的地址信息，第三个参数为对方地址信息长度。

int connect(SOCKET s, const struct sockaddr FAR * name, int namelen);

​ 客户端通过connect函数发起连接请求，如果连接成功，返回0，否则返回SOCKET_ERROR.

int send(SOCKET s, const char FAR *buf, int len, int flags);

​ windows头文件：<WinSock2.h>

​ linux头文件：<sys/socket.h>

​ 客户端与服务端之间通过send函数发送数据给对方，buf为实际要发送的数据， len为数据长度， flags一般填0，同步socket的send流程如下：

  • 比较socket发送缓冲区与len的大小，若len>发送缓冲区的长度，send分段发送
  • 判断socket是否正在发送缓冲区的数据
    • 是，先等待发送中的数据发送完成
    • 否，比较发送缓冲区的剩余长度与len的大小
      • 发送缓冲区的剩余长度 > len，将buf拷贝至缓冲区尾部，返回拷贝字节数
      • 发送缓冲区的剩余长度 < len，等待现有缓冲区数据发送完毕，再将buf拷贝至缓冲区，返回拷贝字节数
  • 出现拷贝失败或者网络断开等问题导致send失败，返回SOCKET_ERROR

int recv(SOCKET s, const char FAR *buf, int len, int flags);

​ windows头文件：<WinSock2.h>

​ linux头文件：<sys/socket.h>

​ 客户端与服务端之间通过recv函数读取对方发送的数据，buf用来接收数据， len为buf长度， flags一般填0，同步socket的recv流程如下：

  • 判断socket是否正在发送数据，如果正在发送则等待数据发送完毕，socket发送数据过程中出现网络断开，返回SOCKET_ERROR

  • socket的发送缓冲区没有数据之后，recv先检查socket的接收缓冲区，如果接收缓冲区内没有数据或者正在接收数据，recv一直等待直到数据接收完毕

  • 缓冲区接收完毕后，recv会将缓冲区数据拷贝至buf中，如果缓冲区中数数据长度>len，则需要多次拷贝，recv返回实际拷贝字节数

  • recv拷贝时出错返回SOCKET_ERROR，网络断开返回0
