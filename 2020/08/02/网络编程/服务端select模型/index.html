服务端select网络模型

John Doe
2020-08-02 (Updated: 2022-01-07)
网络编程
项目笔记
简介

​ 在传统的网络模型中，如果一个服务端与多个客户端进行socket连接，由于accept和recv方法的阻塞性，在服务端等待连接或者等待某一个客户端发送消息时，系统会进入阻塞状态，造成资源浪费。解决这个问题的方法之一，便是采用select网络模型，将多个socket加入监控队列，一旦某个socket有消息到来时，可以随时对其进行recv或者accept处理，减少系统阻塞时间。

​ select模型适用于小用户量访问。

原理与过程

  • 创建3个socket集合（fd_set），fdRead、fdWrite和fdExp，分别用来存放有消息可读、可写入及发生异常的socket

  • 将集合初始化（清空 FD_ZERO(&socketSet)）

  • 将希望select监听的socket放入对应的集合中，比如监听某个socket是否有消息可读，则将其加入fdRead集合中（FD_SET(socket, &fdRead)）

  • 遍历目标socket集合，对于select模型选出的socket进行处理:

    • 服务端socket：一般表示有连接到来，accept
    • 客户端socket：一般表示有消息到来，recv
数据类型与API

fd_set

​ 上述socket集合的数据类型，结构体原型如下：

  • fd_count：select选出的socket个数，如果是fdRead，则表示select监听期间有fd_count个socket有信息过来，可以接受处理
  • fd_array：监听的socket集合，如果是fdRead，则fd_count>0时，索引范围[0，fd_count-1]内均为有消息的socket
  • FD_SETSIZE：集合长度，默认值为64，如果想要更改可以在#include<WINSOCK2.H>之前重新定义该宏

FD_ZERO

​ #define FD_ZERO(set) (((fd_set FAR *)(set))->fd_count=0)

​ 将socket集合中监听到的数量请0，一般用来初始化集合

FD_SET

​ 如果socket不在集合中且集合仍有剩余空间，将socket加入集合

FD_CLR

​ 将socket从集合中移除，一般移出后再closesocket

FD_ISSET

​ #define FD_ISSET(fd, set) __WSAFDIsSet((SOCKET)(fd), (fd_set FAR *)(set))

​ 判断socket是否被select出，比如socket在fdRead集合中，此宏可用来判断该socket是否有消息可读

select

  • nfds：固定填入socket+1

  • readfds、wrietefds、exceptfds：监听的集合

  • timeout：

    等待时间，填入NULL表示一直等待直到select出目标socket

  • 返回值：

    • 0：等待时间内没有找到可读、可写或异常socket
    • >0：找到可读、可写或异常socket
    • SOCKER_ERROR：执行过程中出现异常
