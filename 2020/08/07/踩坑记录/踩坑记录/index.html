socket error 10053错误解决

2020-08-07 (Updated: 2022-02-16)
网络编程
踩坑记录
问题描述

​ 客户端启动发送消息，服务端一直不进行处理，短时间后服务端和客户端均报错，error code为10053。

分析原因

​ 上网百度，winsock对此的描述是：The error can occur when the local network system aborts

a connection. This would occur if WinSock aborts an established connection after data

retransmission fails (receiver never acknowledges data sent on a datastream socket).

​ 看起来是服务端主动中断了连接，那么很有可能是传入recv函数的缓冲区满了，而造成这种问题的原因是服务端的处理速度跟不上发送方的发送速度，但是考虑到缓冲区设置的比较大，而且本身服务端处理逻辑也很简单，不至于来不及处理客户端发送的数据，结合前面服务端就没有处理过任何客户端的消息，但是客户端确实发送成功了，问题很大可能出在拆包的过程中。

问题定位

​ 看了下拆包过程，果然发现问题…

​ 首先看下缓冲区类型：

​ 再来看下拆包过程中对缓冲区是否有完整消息的判断：

​ 在判断缓冲区长度的时候没有用队列尾部索引值而是用strlen来判断缓冲区的长度，如果缓冲区是连续的，尾部索引之前所有位置都不为‘\0’，这样确实没有问题，但问题就在于下面：

​ 客户端在发送的时候，是将其他结构体类型消息强转为char型字节流发送过去，强转的过程中必然导致存在’\0’，这样在用strlen来判断服务端接收缓冲区的长度的话，必然不正确,甚至得到的长度始终为0，即强转过程中缓冲区的开头即为’\0’，这导致服务端始终以为没有消息到来，而不从消息缓冲区中取出消息，导致接受缓冲区溢出。

问题解决

​ 使用缓冲区尾部索引来判断缓冲区长度。
