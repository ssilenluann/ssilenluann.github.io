C++ 内存管理

John Doe
2020-08-15 (Updated: 2022-06-10)
编程语言
C++
简介

​ C++程序做好内存管理，可以避免内存碎片的产生，使程序长期稳定、高效地运行，实现内存管理常见有三种方式：

  • 内存池

    从系统中申请足够大小的内存，由程序自己管理。

  • 对象池

    创建足够多的对象，减少创建和释放对象的消耗。

  • 智能指针

    保证被创建的对象被正确地释放。

对象池

​ 对象池的实现过程与内存池类似，不同之处仅在于每个对象池中存放的都是特定类型的对象。

​ 对象池本身也是一个链表，每个节点包含节点头与对象内存，其分配内存的过程与内存池基本一致，对象池是一个模板类，创建模板类对象时需要指定对象的类型ClasssName及对象池的大小nSize。

再看对象池的管理类，其负责创建对象池，并在创建对象时从对象池中分配内存，其内部重写了new和delete运算符，在运算符内部获取对应对象的对象池并分配或回收内存空间。

因此，需要加入对象池管理的类只需要继承该基类即可。

智能指针

​ c++11提出了三种智能指针，unique_ptr, shared_ptr和weak_ptr，并将auto_ptr摒弃，智能指针可以实现对new出来的对象进行管理，当符合对应的条件时，三种指针会自动析构对象并释放内存。

shared_ptr

​ shared_ptr可以用来管理一个对象的内存，并可以与其他shared_ptr对象共项其管理对象的内存，其内部包含两个主要的指针：

  • 指向对象的指针(stored pointer)
  • 用于控制引用计数的指针（共享）(owned pointer)

​ 当新的shared_ptr参与管理对象内存时，所有对象内部的计数指针计数值都会+1，当shared_ptr对象声明周期结束或者失去其管理身份，计数器的值-1，计数值为0时，最后一个参与管理的shared_ptr对象会释放指向对象的指针。

unique_ptr

其与shared_ptr的不同之处在于unique_ptr无拷贝构造，其资源对象不可与其他unique_ptr共享，只能在unique_ptr对象之间移动，unique_ptr内部有两个组件：

  • stored pointer：指向被管理的对象

  • stored deleter：指向删除器，用于析构被管理的对象

智能指针与对象池的结合

如果想要将某对象内存交由对象池管理，而其生命周期交由智能指针管理，应使用new运算符，从对象池中分配内存，然后将指针交由智能指针管理。
