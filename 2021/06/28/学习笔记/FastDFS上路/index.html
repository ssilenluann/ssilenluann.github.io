1.FastDFS

1.1 FastDFS简介

1.1.1 FastDFS体系结构

​ FastDFS是一个开源的轻量级分布式文件系统，它对文件进行管理，功能包括：文件存储，文件同步，文件访问（上传、下载）等，解决了大容量存储和负载均衡的问题，特别适合以文件为载体的在线服务，如相册网站，视频网站等。

​ FastDFS架构包括Tracker Server和Storage Server,客户端请求Tracker server进行文件访问，tracker server调度Storage server完成用户访问。

  • Tracker Server: 负载均衡和调度，通过Tracker Server可以在文件上传时根据一些策略找到Storage提供文件上传服务，可以理解为追踪或调度服务器
  • Storage Server: 负责文件存储，客户端上传的文件最终存储在Storage服务器上，Storage Server没有实现自己的文件系统而是利用操作系统的文件系统来管理文件，可以理解为存储服务器
1.1.2 工作流程概述

​ 上图为FastDFS的工作流程，首先看Storage Server,其包含了多个服务器组，组内服务器文件同步备份实现高可用，通过增加组可以实现线性扩容，Storage Server定时向Tracker Server注册其组信息，用户提交访问文件的请求后，微服务从Tracker Server总获取Storage信息，并且找到可用的文件存储服务器,返回给微服务项目，最后，微服务将文件上传至对应的storage服务器。

1.1.3 上传流程

​ 图为使用FastDFS上传文件时系统工作流程，客户端上传文件后，存储服务器会将文件ID返回给客户端，此文件ID用于以后访问文件的索引信息，文件索引信息包括：组名，虚拟磁盘路径，数据两级目录及文件名（groupN/M00/01/02/xxx.xx）

组名：文件上传后所在的storage组名称，上传成功后storage服务器返回，客户端自行保存

虚拟磁盘路径：storage server配置的虚拟路径，与磁盘选项store_pathN对应，指向了storage组中的某个节点的具体硬盘地址

两级数据目录：storage服务器在每个虚拟磁盘路径下创建的两级目录，用来存储文件

文件名：与上传时的文件名不同，此处的文件名包含storage服务器ip，文件创建时间戳，文件大小，随机数和文件扩展名等信息

1.2 FastDFS搭建

1.2.1 安装镜像

安装镜像

docker pull morunchang/fastdfs

运行tracker

docker run -d –name tracker –net=host morunchang/fastdfs sh tracker.sh

运行storage（如果需要新增storage服务器，需要再次运行该命令，注意组名）

docker run -d –name storage –net=host -e TRACKER_IP=139.196.79.161:22122 -e GROUP_NAME=group1 morunchang/fastdfs sh storage.sh

1.2.2 配置Nginx

FastDFS的镜像中已经继承了Nginx，所以直接进入容器中就可以进行配置Nginx

docker exec -it storage /bin/bash

查看Nginx配置文件

vi /etc/nginx/conf/nginx.conf

nginx默认监听8080端口，并将/M00开头的URL交由FastDFS处理

设置容器开机自动启动

docker update –restart=always tracker

docker update –restart=always storage
