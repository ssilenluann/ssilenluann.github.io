从媒体接入控制到虚拟局域网

John Doe
2021-07-05 (Updated: 2021-12-09)
计算机网络
数据链路层

1. 简介

​ 对于一个简单的总线型网络拓扑，其中有多个网络节点，如果恰巧有两个或更多的节点在同一时刻发送数据，这就导致信号在总线上会产生碰撞，从而导致信号发送失败，这时，共享信道就要着重考虑一个问题就是如何协调各个发送和接收节点对共享信道的占用，即接入媒体控制MAC（Medium Access Control）。本文将简单介绍用来控制共享信道占用的CSMA/CD协议，以及现在常用的网络交换设备以太网交换机。

2. CSMA/CD

2.1 CSMA/CD

​ CSMA/CD全称为载波监听多址接入/碰撞检测（carrier sense multiple access/collision detection）。

  • 多址接入MA：多个节点连接在一条总线上竞争使用总线
  • 载波监听CS：每个节点发送帧之前先要检测下总线上是否有其他节点在发送帧
    • 若检测到总线空闲96个bit空闲时间，则发送帧
    • 若检测到总线忙，则等待至总线空闲96个bit后发送帧
  • 碰撞检测CD:每一个正在发送帧的节点边发送边检测碰撞
    • 一旦检测出碰撞，则停止发送，等待一段时间后再发送
2.2 争用期（碰撞窗口）

​ 如图示，假设AD之间的单程端到端之间的传播时延为t,在总线空闲时，A想给D发送信息，期间ABCD有可能都会发送信息，那么理论上，最长经过时间2t后，A就能检测到信号碰撞，时间2t称为争用期或碰撞窗口。

因此，经过争用期这段时间内没有检测到碰撞，则这次发生肯定不会产生碰撞，并且争用期的时间是不确定的，端到端往返时延越长，争用期越长，主机越多，发生碰撞的概率越大。

2.3 截断二进制指数退避算法

​ 正如前面所讲，如果一个节点发送帧后检测到了碰撞，那么该节点会等待一段时间后再进行发送，该等待时间由截断二进制指数退避算法计算得来。

​

  • 若连续多次发生碰撞，就有可能表示较多节点参与总线竞争，使用退避算法可使平均等待时间加长，减少碰撞的概率，有利于系统稳定
  • 当重传16次仍不能成功时，表明同时发送帧的节点太多，以至于连续发生碰撞，则丢弃该帧，并向上层报告

​ 需要说明的是，在无线局域网中，不能使用CSMA/CD协议，而是使用CSMD/CA（载波监听多址接入/碰撞避免）协议。

3. 以太网交换机

3.1交换机简介

​ 以太网交换机通常由多个接口，每个接口都可以直接与一台主机或另一个以太网交换机相连，一般工作在全双工模式，其具有并行性，能同时连通多对接口，使多对主机能同时通信，无碰撞（不使用CSMA/CD协议）。

​ 其工作在数据链路层和物理层，收到帧后，在帧交换表中查找帧的目的MAC地址所对应的接口号，然后通过该接口转发该帧，其帧交换表是通过自学习算法自动地逐渐建立起来的。

3.2 对比交换机与集线器
  • 单个节点单播帧传输

​ 当集线器接收到某个节点的单播帧时，仍将其发送到其他主机，需要其他主机自行对比自身MAC地址与帧中的目标MAC地址，然后决定是否接受该帧；而交换机接收到某个节点的单播帧时，会根据帧交换表查找目的MAC地址所在的接口，只向该接口所连接的节点进行转发。

  • 多节点单播帧传输

​ 对于集线器，同时收到多个单播帧，必定会产生信号冲突，然后发送节点会基于CSMA/CD进行避让，而交换机会将接受到的帧先缓存起来，然后逐个发送给目标主机

  • 广播帧传输

    两者一致

  • 拓展网络下传输单播帧

  • 扩展网络下多节点传输单播帧

​ 可以看出，如果使用集线器扩展以太网，不仅扩大了其广播域，还扩大了其碰撞域，而使用交换机扩展以太网，只扩大其广播域，不会扩大其碰撞域

3.3 以太网交换机自学习和转发帧的流程

​ 如图示，两台交换机连接在一起，上面各有4个接口，一共连接ABCDEF6台主机，假设各主机都已经知道其他主机的MAC地址，无需通过ARP协议获取其他主机MAC地址。

现假设A要给B发送信息，则首先A将帧传输至接口1，交换机1会记录下接口1对应A的MAC地址，并将记录填写在帧交换表中，完成登记。

​

​ 登记完成后，交换机1需要将帧进行转发，转发前先从帧交换表中查询帧中目标MAC地址对应的接口，发现查询不到，于是将帧转发至2、3、4接口，进行泛洪。

​ 接下来，主机B、C接收到帧，并根据帧中目标MAC地址与自身MAC地址进行比较，决定是否接受，而从接口4转发的帧会进入交换机2，交换机2和交换机1一样，先进行登记，将A的MAC地址与端口2的对应信息记录至帧交换表中，然后进行经过查询后将帧转发至所有接口，主机DEF会接收到帧然后进行判断是否接收。

​ 如此重复多个回合，交换机就会将所有的接口和MAC地址的对应情况全部记录在帧交换表中。

​ 需要注意的是，每条记录都有自己的有效时间，到期自动删除。

4. 虚拟局域网VLAN

​ 以太网交换机工作在数据链路层（包括物理层），使用一个或多个以太网交换机互联起来的交换式以太网，其所有站点都属于同一个广播域，即以太网中任意站点发送广播帧，其余所有站点均收到广播帧。随着交换式以太网规模的扩大，广播帧出现的就会越来越频繁，造成广播风暴及其他潜在的安全问题，为了解决这一问题，虚拟局域网技术应运而生。

4.1 虚拟局域网VLAN概述

​ 虚拟局域网VLAN（virtual local area network）是一种将局域网内的设备划分为与物理位置无关的逻辑组的技术，这些逻辑组具有某些共同的需求。

​ 比如，可以将图示的局域网划分为两个虚拟局域网VLAN1、VLAN2,VLAN1中的广播帧不会发送至VLAN2中的站点，VLAN2中的广播帧也不会发送至VLAN1中的站点。

4.2 VALN的实现机制
4.2.1 IEEE 802.1Q帧

​ IEEE 802.1Q帧对以太网MAC帧格式进行了扩展，插入了4字节的VLAN标记，如下：

  • VLAN标记的最后12字节称为VLAN标识符VID，它唯一地标志了以太网帧属于哪一个VLAN

  • 802.1Q帧通过交换机进行处理，而非用户主机进行处理

    • 当交换机收到普通的以太网帧时，会插入4字节的VLAN标记使其转变为802.1Q帧，简称“打标签”
    • 当交换机转发802.1Q帧时，可能会去掉4字节的VLAN标记使其变为普通的以太网帧，简称“去标签”
4.2.2 交换机的端口类型

​ 交换机的端口类型有三种：Access、Trunk、Hybrid（思科交换机没有次类型端口）。

Access端口

  • 一般用于连接用户计算机

  • 只属于一个VLAN

  • 端口的PVID（端口VLAN ID）值与端口所属VLAN的ID值相同，默认为1

  • 端口接收帧处理方法：

    一般只接受“未打标签”的普通以太网MAC帧，然后根据自身PVID值给帧“打标签”，字段中VID值与PVID相同

  • 端口发送处理方法：

    若帧中的VID与端口的PVID值相同，则“去标签”并转发该帧，否则不转发

Trunk端口

  • 一般用于交换机之间或交换机与路由器之前的互连

  • 可以属于多个VLAN

  • 用户可以设置Trunk端口中的PVID值，默认情况下Trunk端口的PVID值为1

  • 端口发送处理方法：

    • 对VID等于PVID的帧，“去标签”再转发
    • 对VID不等于PVID的帧，直接转发

  • 端口接收处理方法：

    • 对于“未打标签”的帧，将帧“打标签”
    • “已打标签”的帧，直接接收

    ​ 如上图，交换机1的端口5为Trunk型，广播帧VID与端口PVID不等，直接转发，交换机2的端口5为Trunk型，广播帧“已打标签”，直接接收

    ​ 如上图，交换机1的端口5为Trunk型，广播帧VID与端口PVID相等，“去标签”后转发，交换机2的端口5为Trunk型，广播帧“未打标签”，“打标签”后再接收

Hybrid端口

  • 即可用于交换机之间或交换机与路由器之间的互连（同Trunk端口），也可用于交换机与计算机之间的互连（同Access端口）

  • 可以属于多个VLAN（同Trunk端口）

  • 用户可以设置Hybrid端口的PVID值，默认情况下，Hybrid端口的PVID值为1（同Trunk端口）

  • 发送处理方法：

    查看帧的VID是否在端口的“去标签”列表中：

    • 在，“去标签”后再转发
    • 不在，直接转发

  • 接收处理方法

    • “未打标签”的帧，“打标签”后接收
    • “已打标签”的帧，直接接收
4.2.3 设计一个简单局域网

​ 主机A、B、C连接在同一个交换机上的不同接口，利用Hybrid端口，实现A和B都可以和C通信，而AB之间不能相互通信

​ 此设计的关键之处在于当交换机将帧转发至计算机时，必须先去标签，因为计算机不能处理802.1Q帧，只能处理普通MAC帧，因此，在图示设计中，AB与C都能正常通讯，因为端口3的去标签列表中包含了端口1、2的PVID，即从端口1、2来的帧，经端口3转发后，都会被去除标签，成为正常MAC帧，主机C可以接受，同理，端口3来的帧也会被端口1、2去标签成为正常帧，而被A、B接受，所以AB都可以和C相互通讯。

​ 而端口1的去标签中不包含端口2的PVID，这就导致从端口1中来的数据帧，经由端口2转发后，没有被去标签，仍然是802.1Q帧，计算机B不能识别，计算机B不能接受到A的信息，同理，计算机A也不能接受到B的信息，即AB不能相互通讯。
